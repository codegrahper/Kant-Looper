# GLOSSARY

칸트루퍼 개발·운영 중에 쓰인 용어 사전. 개발자가 아니어도 클로드와 소통할 수
있도록, 코드에는 등장하지만 대화에 직접 드러나지 않는 용어까지 포함한다.
새 용어가 나올 때마다 이 브랜치에 계속 누적한다.

---

**SSOT** : (Single Source of Truth)는 데이터나 정보의 신뢰할 수 있는 기준을
하나의 출처로 통일하여 관리하는 원칙.
> 라우팅 가이드(`references/multimodel-coding-agent-routing-guide.md`)를
> 참고 자료로 삼되, 실제로 무엇을 선택할지는 매번 클로드가 판단한다 — 문서가
> 유일한 근거지만 그걸 기계가 강제로 파싱해서 따르진 않는다.

**ff-only** : (Fast-Forward Only)는 Git에서 브랜치 병합 시 병합 커밋을 만들지
않고, 브랜치 포인터를 앞으로 이동하는 방식만 허용하는 옵션.
> 개발 브랜치(`Kant-looper-branch`)에서 만든 커밋을 배포판
> (`main`)에 반영할 때 `git merge --ff-only`만 쓴다 — 병합 커밋이 안 생기니
> 히스토리가 한 줄로 깔끔하게 유지된다.

**verdict** : 외부 도구(에이전트)가 작업을 마친 뒤 내리는 최종 판정. "완료했다"는
말이 아니라 `PASS`(통과) / `CHANGES_REQUESTED`(수정 필요) / `BLOCKED`(막힘) /
`INVALID_OUTPUT`(판정 자체를 못 냄) 중 하나로 정해진 형식(JSON)으로 나와야 한다.
> codex가 포트폴리오 사이트 리뷰에서 `CHANGES_REQUESTED`를 냈을
> 때, 그 안에 "왜 안 되는지"(agy가 불필요한 로그 파일을 같이 커밋함)까지
> 정확히 담겨 있어서 실제로 유효한 지적이었다.

**worktree** : 하나의 git 저장소를 여러 개의 독립된 작업 폴더로 동시에 열어두는
git 기능. 브랜치가 달라도 같은 `.git` 히스토리를 공유하면서, 실제 파일은 서로
다른 폴더에 격리된다.
> 외부 도구를 부를 때마다 `/tmp/kant-worktree-<번호>` 같은 임시
> 폴더를 새로 만들어 그 안에서만 작업시킨다 — 원본 저장소나 다른 작업에 실수로
> 영향을 주지 않도록 하는 안전장치다.

**fallback** : 1순위로 시도한 것이 실패했을 때 자동으로 대체 수단으로 넘어가는
동작. 칸트루퍼에서는 `fallback-dispatcher.sh`가 이 역할을 한다.
> opencode의 glm-4.7이 verdict를 못 내고 실패하더라도, 이 안전망
> 덕분에 다른 도구/모델로 자동 전환되므로 모델을 미리 빼지 않고 유지하기로
> 결정했다.

**어댑터(adapter)** : 서로 다른 인터페이스를 가진 것들을 같은 방식으로 다룰 수
있게 중간에서 변환해주는 프로그램. 원래 뜻은 전자제품 콘센트 모양을 바꿔주는
그 어댑터와 같다.
> codex/grok/opencode/agy/claude 다섯 개 CLI 도구는 실행 방식이 전부 다른데,
> 각 도구 전용 어댑터 스크립트(`adapter-agy.sh` 등)가 "프롬프트 받아서
> verdict 돌려주기"라는 같은 규격으로 맞춰준다.

**디스패처(dispatcher)** : 들어온 작업이나 이벤트를 보고 어디로 보낼지 결정해서
넘겨주는 역할을 하는 프로그램.
> 에이전트가 작업을 마치면 그 결과를 보고 다음 에이전트에게 자동으로
> 넘기는 "디스패처"를 만들어봤지만, 판정을 스크립트에 맡기는 방식이 계속
> 문제를 일으켜 이 세션에서 걷어내고 클로드가 직접 다음 작업을 판단하는
> 방식으로 되돌렸다.

**콜백(callback)** : 어떤 작업이 끝났을 때 "끝났다"고 자동으로 알려주는 신호나
그 신호를 받아서 실행되는 처리. 사람이 계속 확인하러 갈 필요 없이, 끝나는
쪽에서 먼저 알려준다.
> 백그라운드로 실행한 작업이 끝나면 하네스가 알아서 완료 알림을 보내주는데,
> 이게 콜백이다. 알림을 안 걸어두면(=콜백 없이 그냥 던지기만 하면) 실제로
> 끝났는지 아무도 모르는 상태가 된다 — 오늘 세션에서 실제로 이 실수를 했다.

**백그라운드 실행 / detach** : 명령을 실행시켜 놓고 그 결과를 기다리지 않고
바로 다음 일을 할 수 있게 하는 방식. 반대말은 포그라운드(foreground) —
끝날 때까지 그 자리에서 기다리는 방식.
> `kant-loop.sh run ... --detach`는 외부 도구 호출을 백그라운드로 던져놓고
> 바로 제어권을 돌려준다. 짧은 작업은 그냥 기다려도(foreground) 되고, 오래
> 걸릴 작업만 detach를 쓴다.

**게이트(gate)** : 다음 단계로 넘어가기 전에 반드시 통과해야 하는 자동 검사
관문.
> 외부 도구가 코드를 고친 뒤, 실제로 git에 변경사항이 있는지·안전 규칙을
> 어기지 않았는지 같은 게이트를 전부 통과해야만 커밋까지 이어진다. 하나라도
> 막히면 그 자리에서 실패로 처리된다.

**샌드박스(sandbox)** : 실제 환경에 영향을 주지 못하게 격리된, 제한된 실행
공간. 어린이 놀이터의 모래상자에서 따온 말 — 안에서 뭘 해도 바깥은 안전하다.
> agy를 부를 때 "읽기 전용"과 "쓰기 허용" 두 가지 샌드박스 모드가 있다.
> 리뷰할 때는 읽기 전용으로 묶어서, 실수로 파일을 고치는 사고를 막는다.

**MCP** : (Model Context Protocol)은 AI 모델이 외부 도구·데이터에 정해진
방식으로 접근할 수 있게 하는 공통 규격.
> agy는 Stitch(Google의 UI 디자인 생성 도구)에 MCP로 이미 연결돼 있지만,
> 프롬프트에서 먼저 호출하라고 시키지 않으면 그냥 넘어간다는 걸 오늘
> 세션에서 확인했다.

**HPRAR** : Plan(계획)→Implement(구현)→Review(검토)→Repair(수정)를 사람 개입
없이 자동으로 반복시키려던, 지금은 포기한 실행 방식의 이름.
> "자동으로 계속 도는 루프"를 여러 번 시도했지만 매번 같은 지점(판정을
> 기계에 맡기는 부분)에서 무너져서, 이 방식 자체를 포기하고 클로드가 매
> 단계 직접 판단하는 구조로 바꿨다.

**토큰(token)** : AI 모델이 글을 읽고 쓸 때 계산하는 최소 단위이자, 그만큼
쓴 양에 따라 비용·사용량이 매겨지는 기준.
> 클로드의 사용량은 제한적이라, 직접 코드를 다 짜기보다 다른 도구에
> 위임해서 클로드는 판단·검증에만 토큰을 쓰는 게 칸트루퍼가 존재하는
> 이유 중 하나다.

**브랜치(branch)** : 하나의 저장소 안에서 원본에 영향을 주지 않고 독립적으로
작업할 수 있게 나눠놓은 작업 줄기.
> 외부 도구가 작업할 때마다 `agent/kant/<작업이름>` 같은 새 브랜치를 만들어
> 그 안에서만 커밋한다 — main 브랜치는 항상 깨끗하게 유지된다.

**diff** : 변경 전과 변경 후를 비교해서 "무엇이 달라졌는지"만 뽑아 보여주는 것.
> 에이전트가 "다 고쳤다"고 말해도 그 말만 믿지 않고, 실제로 git diff에
> 변경 내용이 있는지 항상 대조해서 확인한다.
